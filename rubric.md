# Grdaing:
```
Total marks	 		= 	25 %
It is divided as follows:
Presentation  		= 	8%
Code submission 	= 	8%
Report			= 	9%
```

# Presentation Guidelines:

```
10 minutes presentation + 5 mins Q&A
Should clearly state title, motivation, objectives, methodology, results and your own analysis / conclusions. 
Avoid putting code snapshots on the slides, unless necessary to discuss the results. 
Try to create visualizations e.g. plots, figures, diagrams to show the results instead of long tables if data is large.
Keep your presentation strictly within time limits. 
```

Students will upload the report on GradeScope as their official submission before the deadline. 

# Report Submission Guidelines
```
1. The report must have the github repo link to connect with your code submission.

2. The main part of the report should be ~5-7 pages (NOT including figures, which means you can have more pages in total
if you account for figures space) - single column, 11 pt font, 1 inch margin on all sides. It is NOT recommended to submit
a report that has >7 pages (do not count the space for figures)as the main part of the report.

3. Please make good use of the appendix / supporting information section. Please write your main part of the report concisely
(5-7 pages) and feel free to put extrainformation, extra figures, large tables, detailed theory etc. in the appendices / supporting
information at the end (no page limit).

4. You must write a "contribution" section explaining the contribution of each team member to this project. It doesn't need to be
detailed, 2-5 lines for each person's contribution will be good enough. 

5. Your github repo link must be noted down in the report (add it in the supporting information section - you can see it in any
published article to get an idea how they do it). 

6. You will submit this report on the gradescope. However, your report and presentation slides must be included in your github repo
as well. So technically, your github repo will contain everything regarding your final project. 

7. You can use the following template for your report but feel free to choose your own as well:

- Abstract (4-8 lines)
- Background (including introduction and motivation)
- Objectives - define your objectives very clearly
- Methodology
- Results and Discussion
- Conclusions - discuss how you achieved your objectives
- Future work
- Contribution of each team member
- References
- Appendices / Supporting Information
```


# Final Code - Github Submission
```
1. The github repository will work as your comprehensive submission for the final project. 
Github repo must contain fully working code, presentation slides and report pdf - all together. 

2. README file must contain simple instructions to follow to run the code to reproduce your results.
If there is a need to install the docker file, all commands should be written step by step. 

3. It will be ideal to create a run.sh file in the main directory that anyone can execute to
(a) create the required environment / docker, (2) run the code, (3) analyze the results. 

```
